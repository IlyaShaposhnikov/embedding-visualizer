# Embedding Playground

**Интерактивный инструмент для исследования эмбеддингов (векторных представлений) слов: Word2Vec, GloVe, поиск ближайших соседей, поиск аналогий и 2D-визуализация.**


## Возможности

- **Автоматическая загрузка** предобученных моделей с проверкой целостности:
  - Word2Vec GoogleNews (3 млн слов, 300-мерные векторы)
  - GloVe 6B (400 тыс. слов, 50/100/200/300-мерные векторы)
- **Оптимизированное кэширование** — модели сохраняются в бинарном формате Gensim для мгновенной загрузки при повторных запусках
- **Поиск ближайших соседей** — нахождение семантически близких слов с визуальными шкалами сходства
- **Поиск аналогий** — решение задач вида `king - man + woman = ?` с 2D-визуализацией векторных отношений (флаг `-v`) с помощью PCA/t-SNE
- **Семантические кластеры** — 2D-визуализация векторных отношений опорных слов и их соседей с помощью PCA/t-SNE
- **Оценка качества модели** — тестирование на наборе Google Analogy Test Set (19 544 задачи) с разбивкой по семантическим и синтаксическим группам
- **Интерактивный CLI** — интуитивная командная оболочка с контекстной справкой и демо-режимом


## Быстрый старт

### 1. Клонирование и установка

```bash
git clone https://github.com/IlyaShaposhnikov/embedding-playground.git
cd embedding-playground
python -m venv .venv
source venv/Scripts/activate  #Windows
source .venv/bin/activate  # Linux/macOS
pip install -r requirements.txt
```

### 2. Запуск интерактивной оболочки

```bash
python main.py
```

При первом запуске модели загрузятся автоматически (~1,5 ГБ для Word2Vec, ~800 МБ для GloVe). Последующие запуски будут использовать кэшированные бинарные файлы.

### 3. Примеры команд

```bash
>>> nn king 5                       # 5 ближайших соседей слова «king»
>>> ana paris france berlin -v tsne # paris - france = ? - berlin + визуализация (t-SNE)
>>> vc king queen computer 3 pca    # визуализациях кластеров для 3 опорных слов, по 3 соседа (PCA)
>>> eval                            # оценка качества модели на наборе Google Analogy Test Set
>>> demo                            # полная демонстрация (соседи, аналогии, кластеры)
```


## Справочник команд

| Команда | Описание |
|---------|----------|
| `use <model>` | Переключить активную модель: `word2vec` или `glove` |
| `nn <word> [topn]` | Ближайшие соседи (по умолчанию `topn=5`) |
| `ana <w1> <w2> <w3> [topn] [-v] [m]` | Поиск аналогий `w1 - w2 = ? - w3`<br>• `-v`: визуализация векторных отношений<br>• `[m]`: метод `pca` или `tsne` (по умолчанию `pca`)<br>- **PCA** — _линейная проекция_, сохраняющая **глобальную структуру** путем отображения векторов на оси максимальной дисперсии. Быстро, но может размывать локальные кластеры.<br>- **t-SNE** — _нелинейная проекция_, сохраняющая **локальную структуру данных** путем моделирования попарных сходств. Медленнее, но выявляет кластеры с высокой детализацией ценой искажения глобальной геометрии.<br>→ **Автоматически сохраняется** в `data/visualizations/` |
| `vc <w1> [w2 ...] [n] [m]` | Визуализация семантических кластеров:<br>• Опорные слова: `w1`... (минимум 1)<br>• `[n]`: число соседей для каждого опорного слова (по умолчанию 3, макс. 20)<br>• `[m]`: метод `pca` или `tsne` (по умолчанию `pca`)<br>→ **Автоматически сохраняется** в `data/visualizations/` |
| `eval` | Оценка текущей модели на наборе Google Analogy Test Set |
| `model` | Информация о модели (размер словаря, размерность, использование памяти) |
| `demo` | Полная демонстрация (поиск ближайших соседей, поиск аналогий, визуализация кластеров для обеих моделей) |
| `help` | Справка |
| `exit` / `quit` | Выход из программы |

## Устранение проблем

### Превышена квота Google Drive
Загрузка Word2Vec может завершиться ошибкой:
```
Too many users have viewed or downloaded this file recently...
```
**Решение:**
[Скачайте модель вручную](https://github.com/mmihaltz/word2vec-GoogleNews-vectors/raw/master/GoogleNews-vectors-negative300.bin.gz)

### Низкая точность GloVe 6B.100d
Модель GloVe 6B.100d имеет ограниченный словарь (~400 тыс. слов) и использует 100-мерные векторы:
- **Географические наименования часто отсутствуют** → некоторые семантические разделы пропускаются при оценке (`eval`)
- **Ожидаемая точность**: ~2-5% общая (против 65-75% у Word2Vec GoogleNews 300d)
- **Рекомендация**: для серьезной оценки используйте Word2Vec; GloVe 100d подходит для демонстраций и образовательных целей

### Требования к памяти
| Модель | Требуемая ОЗУ | Первичная загрузка | Загрузка из кэша |
|--------|---------------|--------------------|------------------|
| Word2Vec GoogleNews | ~4,2 ГБ | 3-5 мин | ~10 сек |
| GloVe 6B.100d | ~1,2 ГБ | 1-2 мин | ~5 сек |

Для комфортной работы с обеими моделями рекомендуется ≥6 ГБ свободной оперативной памяти.


## Источники и ссылки

### Основные статьи
- Mikolov, T., Chen, K., Corrado, G., & Dean, J. (2013). [Efficient Estimation of Word Representations in Vector Space](https://arxiv.org/abs/1301.3781). *ICLR*.

### Наборы данных
- **Word2Vec GoogleNews**: обучен на корпусе Google News (3 млрд слов), словарь 3 млн, 300-мерные векторы
- **GloVe 6B**: обучен на Wikipedia 2014 + Gigaword 5 (6 млрд токенов), словарь 400 тыс.
- **Google Analogy Test Set**: 19 544 задачи (8 869 семантических + 10 675 синтаксических)
